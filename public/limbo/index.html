#2 - Data Validation

JSON Schema

  • Declarative way to describe the data we expect to send and receive

  • Can be used to:

    • Verify that data sent to Kasper is valid
    • Generate validation rules for the client-side
    • Generate <form> HTML templates
    • Generate fixtures for unit testing

Routing Validation

  • Routes are now validated when the server is started to check

    • That the route name does not override a reserved path
    • That the route controller exists
    • That the route controller callback exists

Template Validation

  • Server-side templates are validated when the server is started

  • Component client-side templates are validated when the component is packaged

Configuration Validation

  • App configuration file is validated using the Convict module

  • Syntax errors only will raise an exception - this will be extended in future
